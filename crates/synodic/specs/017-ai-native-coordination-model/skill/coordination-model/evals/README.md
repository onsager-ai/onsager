# Coordination Model Behavioral Evals

These evals test whether an AI coding agent loaded with the coordination SKILL.md
actually **exhibits coordination behavior** — not just produces schema-valid YAML,
but thinks and works using the primitives (fork, observe, converge, prune, merge).

## How to run

Each eval is a self-contained prompt you give to an AI coding agent (Claude Code,
GitHub Copilot, etc.) with the coordination skill loaded. The agent produces output
artifacts. The grading script checks structural evidence of coordination.

```bash
# 1. Give the agent a task prompt (from prompts/)
# 2. Collect the agent's output into results/<eval-name>/
# 3. Grade it:
python evals/grade.py results/<eval-name>/
```

## What's tested

| Eval | Primitive | What we check |
|------|-----------|---------------|
| `speculative-swarm-ratelimiter` | Speculative swarm | ≥3 divergent strategies explored, comparison performed, fragments merged |
| `generative-adversarial-api-review` | Generative-adversarial | ≥2 escalation rounds, critic finds real issues, generator fixes them |
| `fractal-decomposition-parser` | Fractal decomposition | Problem split into ≥3 scoped sub-problems, sub-solutions reunified |
| `stigmergic-maintenance-drift` | Stigmergic | Artifact change detected, reactive patch produced, no storm |
| `context-mesh-research` | Context mesh | Knowledge gaps identified, gap-filling spawned, discoveries merged |
| `explore-harden-maintain-e2e` | Composed playbook | All 3 stages evidenced in sequence |

## Grading criteria

Each eval checks for **structural markers** in the agent's output — evidence that
the coordination primitive was followed, not just the final answer quality:

- **fork_evidence**: Agent explicitly produced ≥N divergent approaches/strategies
- **observe_evidence**: Agent compared or evaluated the divergent outputs
- **convergence_evidence**: Agent measured similarity or quality across branches
- **prune_evidence**: Agent explicitly eliminated weaker approaches with reasoning
- **merge_evidence**: Agent combined fragments from multiple approaches into final output
- **escalation_evidence**: Multi-round quality improvement with increasing difficulty
- **decomposition_evidence**: Problem explicitly split into scoped sub-problems
- **reactive_evidence**: Change detection triggered targeted response

These are **structural** checks, not quality judgments — we test process, not polish.

## Output format

Agent output should go in `results/<eval-name>/` with these files:

```
results/<eval-name>/
├── trace.md          # Agent's reasoning trace (copy from chat/terminal)
├── output/           # Any files the agent produced
│   └── ...
└── metadata.json     # Auto-generated by grade.py
```
